Website Chatbot

This projected aimed to:

  • Produce a tool that can answer users’ questions on my personal portfolio website.
  • Select the most appropriate response to questions using machine learning techniques.
  • Use natural language packages to find the stem of user inputs.
  • Integrate this onto the website so it can run synchronously with the webpage.


    • Introduction

    Chat bots are used frequently on websites to assist users who need to ask a question. In my case, my portfolio website would benefit with having a chatbot that is able to answer FAQs from users. This would help me save time in the long-term as recruiters could ask relevant questions to the bot rather than myself.

    Building the training set

    The training set was created using a JSON file, with each specific hypothetical question category being given a ‘tag’. Each category was given a list of potential questions that could be asked by the user, and a series of answers to these questions. In some instances, one answer would be needed as it is a factual answer, whereas in others several answers were given and one would be chosen at random.



    Each question was then processed so that they could be used in the machine learning model. Each sentence was split into a list, and the stem of each word was found. This is so that the words can be compared easily. A reference list was then made from all the words processed, with each item referring to a singular question. A label was created for each question detailing its tag.

    Building the model

    A neural network model was then made of two tiers of 8 nodes. This would be a regression model and therefore be able to provide a probability of a question relating to a specific tag. It was made using TensorFlow’s neural net module, which can be implemented efficiently. The training set was passed through this model, with the resultant model ready to be used on actual data.


    Implementing the bot

    This model is deployed every time a question is asked and returns the tag of the category that it is most probable to belong to. It was integrated into the website via a popup window that had a text-display and text-input field. To allow the bot to run synchronously with the server, Django Channels was used. This provided the capability for the bot to be connected via websockets to an asynchronous layer beneath Django.
