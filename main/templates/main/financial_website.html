Financial Analysis Web Application

  • Website built using Django that allows users to perform statistical analysis on companies in the FTSE 100.
  • Includes candlestick graphs, MACD (Moving Average Convergence Divergence), SMA (Simple Moving Average) and RSI (Relative Strength Index) of FTSE 100 index over the past year.
  • Includes stock price, MACD, RSI and SMA for all stocks in the FTSE 100 currently, and lists current stock statistics.
  • Allows users to register and track any changes they made, with their data being stored in an SQLite database.
  • Allows users to simulate the buying and selling of stocks to track performance.
  • Allows users to build a portfolio to keep track on stocks of interest.
  • Allows the user to screener stocks by using a filtering tool.

Introduction

The stock market appears to be a daunting place for an amateur investor, especially given the effects of the Coronavirus on the current economic climate. An online tool that would be able to guide these types of investors to reasonable decisions or at least to educate them on investing would be practical. Therefore, I have created a webpage that is able to allow users to research and track potential investments. It comprises of:
  • The overall web framework made using Django (Python)
  • Dynamic elements using JavaScript
  • Visual elements using HTML and CSS
  • A database using SQLite
This application would be able to perform the same tasks as the financial GUI package, however this time it will be able to track user data.

User Interaction

The application’s initial configuration allows user access to the homepage, the registration page and the login page. The registration page requires the input of a username, email and password. The validity of these details is assessed and are added to the database. As soon as the user has registered, it logs into their account and notifies them by displaying a ‘toast’. The user can logout by pressing the ‘logout’ tab which sends them back to the initial configuration. If they want to login again, they just need to press the ‘login’ tab and enter their username and password.



Homepage and stock pages

The homepage displays the candlestick, SMA, MACD and RSI plots of the FTSE 100 index. These plots are generated from current index prices parsed from Yahoo Finance. This data and the plots are updated daily by using a background-task manager and scheduling a Cron task.





The stock pages display the stock statistics, candlestick, MACD, SMA and RSI plots for all companies in the FTSE 100. The statistics and stock prices are parsed again from Yahoo Finance and updated daily. The plots are created from these stock prices and are updated daily.



Stock Screener

The stock screener page allows for users to view stocks based on input parameters (currently maximum and minimum). It allows for the user to append an endless list of requirements, and the screener is programmed to select the companies that meet these requirements. The resultant table is interactive so that the row selected transports the user to the respective ticker’s page.



Portfolio

The portfolio page allows for users to build a list of company tickers that is unique to that user. Like the screener, the table is interactive and transports the use rto the respective ticker page when selected.



Simulation

The simulation page tracks the performance in the market of a list of investments selected by the user. The page gathers data on the stock price, increase in price and percentage yield from each stock since the date the user had placed the investment. Using this feature the user can track their portfolio over time, and can test their investment theories out in practice.



Conclusion and Improvements

The overall layout and features of the website accomplishes the main objectives, however there is large scope for improvement:
  • Improve the design of the website and the aesthetics
  • Improve the security so user data cannot be hacked/accessed
  • Add improved stock screener capabilities with use of algorithms
  • Import data for companies in other indexes (Midcap/Smallcap)
  • Allow users to communicate with the admin through a chat option or email
  • Add an automated feature to respond to queries (ie a chat bot)
