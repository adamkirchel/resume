Food Tracking Website Application

This website application allows the user to:

  • Log the contents of their kitchen in a database that can be viewed.
  • Browse recipes from the Food Network that contains their ingredients.
  • Browse other recipes that are grouped by cuisine in the ‘kitchen’ feature.
  • View the ingredients list and method for each individual recipe.
  • Create a timetable for meals in the coming month.
  • Automatically create a shopping list based on what items they have logged onto the database and what they need to create their scheduled meals.
To make the website, I used:

  • BeautifulSoup and Pandas to parse data from the Food Network.
  • Django to build the backend of the website.
  • HTML and CSS to build the front end.
  • JavaScript to build the functionality of elements such as carousels and the navigation bar.
  • SQLite database to store user data and the recipes.
  • Natural Language Processing toolbox to create a recipe search tool.

Introduction

Modern cooking can be boring sometimes, with recipes being used again and again. I decided to create a website application to bring some creativity back into the kitchen whilst making the whole process a lot more stress-free. This application would be able to advise users on what recipes they could make based on what ingredients they had. It also would allow users to schedule meals for the coming month and automatically produce a shopping list based on these meals.

Webscraping with BeautifulSoup

I chose the ‘Food Network’ website as a good starting point to build a database of recipes for the user. I decided to parse all recipes that could be accessed from the main menu, which was around 500. I did this by writing a script using BeautifulSoup that was able to create a url tree from the main menu and obtain a list of urls for all recipes. For each URL I extracted the title, ingredients, method, image, and other details, and uploaded this data to the database.

Logging and processing of ingredients

A feature of the website allows the user to add items to the database using a form. They can log the item, the amount, its price, and its category. These are all displayed in a table, with tabs able to display the different categories.



This list of ingredients is put through a natural language algorithm, which stems the words and compares them against the list of ingredients for each recipe stored in the database. The recipes are then ordered by how many words they share with the ingredients list. The top recipes from this list are then displayed in a carousel at the bottom of the webpage. This carousel was created from scratch using JavaScript. When clicked on, the images link you to the recipe page that displays the ingredients and method.



Meal Planning

The individual recipe pages allow you to schedule a day for you to prepare the meal. This form is submitted and saved on the database, where it is displayed in the calendar under ‘meal plan’. The ingredients from the list of these meals are compared against the ingredients you have logged, and a shopping list is created that excludes these items.
